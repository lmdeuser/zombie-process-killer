# 🧟 Zombie Process Killer

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Shell Script](https://img.shields.io/badge/Shell-Bash-green.svg)](https://www.gnu.org/software/bash/)

Продвинутый скрипт для поиска и безопасного удаления зомби-процессов в Linux. Автоматически обнаруживает процессы-зомби и предлагает различные стратегии их очистки.

## ✨ Возможности

- 🔍 Автоматическое обнаружение зомби-процессов
- 🛡️ Безопасная обработка с подтверждением действий
- 🤖 Автоматический режим для CI/CD и серверов
- 📝 Детальное логирование всех действий
- 🎨 Цветной вывод для лучшей читаемости
- 🚫 Защита системных процессов (PID 1, 2)
- 📊 Мониторинг и статистика

## 📋 Требования

- Linux (ядро 2.6+)
- Bash 4.0+
- Права root (для завершения процессов)

## 🚀 Установка

```bash
# Клонирование репозитория
git clone https://github.com/yourusername/zombie-process-killer.git
cd zombie-process-killer

# Делаем скрипт исполняемым
chmod +x zombie_killer.sh

# (Опционально) Установка в систему
sudo cp zombie_killer.sh /usr/local/bin/zombie-killer
```

## 💻 Использование

Основное использование:

```bash
sudo ./zombie_killer.sh
```

Автоматический режим (без запросов):

```bash
sudo ./zombie_killer.sh --auto
```

Только подсчет зомби-процессов:

```bash
sudo ./zombie_killer.sh --count
```

Просмотр лога операций:

```bash
sudo ./zombie_killer.sh --log
```

Справка:

```bash
./zombie_killer.sh --help
```

## ⚙️ Как это работает

1. **Обнаружение**: Скрипт сканирует все процессы и находит те, у которых статус `Z` (zombie)

2. **Мягкая очистка**: Отправляет сигнал SIGCHLD родительским процессам, чтобы они "забрали" зомби

3. **Проверка**: Если зомби остались, предлагает завершить родительский процесс

4. **Безопасность**: Системные процессы (PID 1, 2) не затрагиваются

5. **Логирование**: Все действия записываются в `/var/log/zombie_killer.log`

## 🔧 Конфигурация

Вы можете изменить следующие параметры в начале скрипта:

```bash
LOG_FILE="/var/log/zombie_killer.log"  # Путь к лог-файлу
AUTO_MODE=false                          # Режим по умолчанию
MAX_RETRIES=3                            # Максимальное количество попыток
TIMEOUT=30                               # Время ожидания в секундах
```

## 🛡️ Безопасность

✅ Скрипт требует root-прав для завершения процессов

✅ Системные процессы (init, kernel) защищены от завершения

✅ В интерактивном режиме требует подтверждения перед каждым опасным действием

✅ Все действия логируются для аудита
